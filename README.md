# Gallery Dashboard Project

## MongoDB Schema

Database: **`events_db`** (configurable via `MONGO_DB_NAME`)

---

### Collection: `new-events`

Each document represents an image processing event. The `sourceId` field references a document in the `sources` collection.

| Field | Type | Required | Description |
|---|---|---|---|
| `imageId` | string | yes | Unique image identifier (e.g. `"img_1234567"`) |
| `personId` | string | yes | Associated person ID (e.g. `"person_12345"`) |
| `imageHash` | string | no | Hash of the image |
| `format` | string | no | Image format (e.g. `"JPEG"`, `"PNG"`, `"BMP"`, `"TIFF"`) |
| `sourceId` | number | yes | Foreign key to `sources.sourceId` |
| `semiSource` | string | no | e.g. `"primary"`, `"secondary"`, `"tertiary"` |
| `createdDate` | Int64 | yes | Unix timestamp in seconds |
| `receivedDate` | Int64 | yes | Unix timestamp in seconds |
| `isFailed` | boolean | no | `true` if processing failed |
| `amountOfFaces` | number | no | Number of faces detected |
| `failureReason` | string | no | e.g. `"IMAGE_TOO_SMALL"`, `"LOW_QUALITY"`, `"NO_FACE_DETECTED"`, `"HASH_COLLISION"`, `"TIMEOUT"` |
| `pendingRestart` | boolean | no | `true` when failed and `restartCount < 3` |
| `restartCount` | number | no | Number of restart attempts made |
| `facesPositions` | array | no | Array of bounding box objects (see below) |

Each entry in `facesPositions`:

| Field | Type | Description |
|---|---|---|
| `left` | number | Left boundary (px) |
| `right` | number | Right boundary (px) |
| `top` | number | Top boundary (px) |
| `bottom` | number | Bottom boundary (px) |

#### Recommended indexes

```js
db.getCollection("new-events").createIndex({ sourceId: 1 })
db.getCollection("new-events").createIndex({ isFailed: 1 })
db.getCollection("new-events").createIndex({ format: 1 })
db.getCollection("new-events").createIndex({ pendingRestart: 1 })
db.getCollection("new-events").createIndex({ receivedDate: -1 })
db.getCollection("new-events").createIndex({ createdDate: 1 })
```

#### Example document

```json
{
  "imageId": "img_1234567",
  "personId": "person_12345",
  "imageHash": "hash_123456789",
  "format": "JPEG",
  "sourceId": 1,
  "semiSource": "primary",
  "createdDate": 1744120200,
  "receivedDate": 1744120245,
  "isFailed": true,
  "amountOfFaces": 2,
  "failureReason": "LOW_QUALITY",
  "pendingRestart": true,
  "restartCount": 1,
  "facesPositions": [
    { "left": 50, "right": 200, "top": 30, "bottom": 180 },
    { "left": 300, "right": 450, "top": 40, "bottom": 190 }
  ]
}
```

### Collection: `sources`

Reference/lookup table for source systems. Values are read as-is from the DB.

| Field | Type | Required | Description |
|---|---|---|---|
| `sourceId` | number | yes | Unique source identifier |
| `SourceName` | string | yes | Source name (referenced by events via `sourceId`) |
| `SourceHebName` | string | yes | Hebrew display name |

#### Example document

```json
{
  "sourceId": 1,
  "SourceName": "louvre",
  "SourceHebName": "לובר"
}
```

---

### Important notes

- **Dates are unix timestamps (Int64, seconds)** — not ISO strings or BSON Date. The server converts to milliseconds for aggregation display.
- `sourceId` in `new-events` is a numeric FK to `sources.sourceId` — the server returns raw `sourceId` and the client resolves it to `SourceName` for display.
- The `_id` field is auto-generated by MongoDB and excluded from API responses.
- Collection names are configurable via `EVENTS_COLL` and `SOURCES_COLL` env vars.
